# 要求内容

## 概要

カレンダー画面で「公開月間 API から取れない日」を**仮スケジュール**で埋め、予約予定（Intent）を登録可能にする。これにより、祝日例外日（datekb=3）や翌月未配信日でも単発の先行登録ができるようになる。併せて「休館」バッジを廃止し、金曜定休（datekb=1）は空欄扱いにする。

## 背景

- カレンダーは予約開放範囲（今日〜+6 日）内は reserve API、範囲外は公開月間 API を使う。
- 公開月間 API は `pims_closed.datekb` に応じて日付をスキップする。具体的には `datekb=1`（店舗定休、府中店だと金曜）と `datekb=3`（GW・祝日等の特別日）を両方とも休館扱いにして `closed_dates` に入れており、UI では「休館」バッジを出して予約予定の登録もできない状態。
- しかし実運用上、`datekb=3` の日は reserve API が開放日になれば実際にはレッスンが行われる。例: 2026-04-29（昭和の日）は公開月間 API では datekb=3 だが、9:00 開放時点で通常通りレッスンが出る。現状は UI が「休館」を出してしまい、ユーザーが予約予定を事前登録できない。
- 翌月未配信問題もある。公開月間 API は月初まで翌月分を返さないため、5 月 1 日以降を表示しようとしても空のまま。定期予約では取れるが単発の先行登録ができない。
- ユーザーとの対話で「わざわざ休館と書かなくていい」「空欄で十分」「過去の同曜日のレッスンを仮で埋めて、予約予定を登録できるようにしたい」との合意を得た。

## 実装対象

### 1. 「休館」バッジの廃止

予約画面カレンダーの日付ヘッダから「休館」バッジを削除し、当該日のセルを空欄で表示する。

受入条件:

- [ ] `datekb=1`（金曜定休）の日に「休館」文字が出ない
- [ ] `datekb=3`（祝日特別日）の日に「休館」文字が出ない
- [ ] 曜日ラベル・日付数字は従来通り表示される
- [ ] 過去の week を表示したとき、従来「休館」が出ていた日が空欄または仮スケジュール表示に置き換わっている

### 2. 仮スケジュール補完

公開月間 API から取得できない日（`datekb=3` で欠損した日、および週末の翌月未配信日など）に、過去の同曜日の観測済みレッスンを「仮」として表示する。

受入条件:

- [ ] `datekb=3` の日（例 2026-04-29）のセルに、1 週前の同曜日（2026-04-22）のレッスンが仮として並ぶ。1 週前に観測がなければ 2 週前、それも無ければ 3 週前まで遡る
- [ ] 翌月未配信日（例 2026-05-01 以降）も同じルールで埋まる
- [ ] 3 週前まで遡っても観測が一切ない場合は、その日のセルは空欄のまま
- [ ] 仮スケジュール由来のレッスンには「仮」バッジが付き、薄いグレーで表示されて他と識別できる
- [ ] 仮スケジュール由来のレッスンは右パネルで予約予定（Intent）登録フォームが出る
- [ ] 9:00 に実データが入った後にカレンダーを再描画すると、仮スケジュールは消えて実データに置き換わる（仮データが残り続けない）
- [ ] 過去の日付（`lesson_date < today`）は仮スケジュール対象外（実績があった日は空のままにする）

### 3. 金曜定休と祝日例外の扱い分け

内部では `datekb=1`（定休）と `datekb=3`（祝日等）を区別する。UI はどちらも「休館」と書かないが、仮スケジュール対象の判定では両者を別扱いにする。

受入条件:

- [ ] `datekb=1`（店舗定休）は仮スケジュール補完の対象外（店舗が閉まっている日に仮のレッスンを出さない）
- [ ] `datekb=1` の日は空欄のまま。Intent 登録フォームも出さない
- [ ] `datekb=3` の日は仮スケジュール補完の対象
- [ ] 公開月間 API のレスポンス解釈で `datekb=1` と `datekb=3` を区別するロジックが入っている

### 4. Lesson エンティティへのマーカー追加

仮スケジュール由来の lesson と通常の `release_pending`（9:00 開放前の最新日）を内部的に区別する。UI 上ではどちらも Intent 登録対象として扱うため見た目は近いが、バッジで差別化する。

受入条件:

- [ ] `Lesson` dataclass に `tentative_source: date | None` フィールドが追加されている（仮データの出典日が入る。実データは `None`）
- [ ] 仮スケジュール lesson は `release_pending=True` かつ `tentative_source=<出典日>` の両方が立つ
- [ ] 既存の release_pending のみの lesson（今日+6 日の未開放枠）は従来通り `tentative_source=None`
- [ ] UI は `tentative_source` が入っているかどうかで「仮」バッジを出し分ける

## スコープ外

- スケジューラによる仮スケジュールの事前生成・キャッシュ（リクエスト時に `observed_lessons` から動的に組み立てる方式で十分）
- 4 週前以上遡っての仮スケジュール補完（3 週前までで打ち切り）
- `datekb` のさらなる細分（`datekb=2` など他の値が観測された場合は従来通り "0 以外は休館扱い" を維持する）
- 仮スケジュール生成の自動テスト以外の統合テスト（既存方針に従いユニットテストはスコープ外）
- 仮スケジュールの店舗（studio）間コピー（同一店舗内の同曜日だけを使う）
