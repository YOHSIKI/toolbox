## ドメイン用語

### カレンダー UI（calendar UI）

月間テンプレートと直近 1 週間の実スケジュールを重ねて、曜日×時刻のマスで表示する画面。予約画面の単発予約タブで使う。

### 希望席（seat preference）

定期予約が実行時に試みる座席番号の優先順位付き配列。第 1 希望から順に試み、取れたらそこで止まる。

### 現在の予約（current reservations）

予約 API から取得した、自分が現在確保している予約の一覧。ダッシュボードに表示する。

### 公開月間 API（public monthly API）

店舗の月間スケジュールを JSONP 形式で返す外部サイトのエンドポイント。代行反映前のテンプレートを提供する。

### 座席番号（seat number / no）

レッスンごとに割り当てられた個別の席識別子。座席レイアウトによって範囲が異なる。

### 座席レイアウト（seat layout / studio room space）

同じスタジオルーム内で複数定義される座席配置（35 名・20 名・25 名など）。レッスンごとにどのレイアウトを使うかが切り替わる。

### 実スケジュール（actual schedule）

予約 API から取得した直近 1 週間分のレッスン一覧。代行や時間変更が反映された状態。

### 単発予約（one-shot reservation）

カレンダーから直接 1 件のレッスンを予約する操作。予約画面の単発予約タブで行う。

### 代行（instructor substitution）

通常とは別の担当者がレッスンを受け持つこと。週によって発生しうるため、定期予約のキーには含めない。UI では「代行」バッジで差分を示す。

### この回だけ変更（one-time override）

定期予約で取れた予約について、その回に限り席を別に変更する操作。定期予約のルール自体は変わらず、翌週以降の自動予約には影響しない。

### ダッシュボード（dashboard）

ログイン直後に表示する画面。当日サマリー、現在の予約、予約予定、予約履歴を 1 画面にまとめる。

### 定期予約（recurring reservation）

毎週決まった曜日・時刻のレッスンを自動で予約する仕組み。曜日・時刻・プログラム ID・希望席の配列などをキーに定義する。

### 店舗（studio）

セントラルスポーツのクラブを表す単位。識別子は `studio_id`。

### 店舗ルーム（studio room）

店舗内のスタジオ・プール・ジム等を区別する単位。識別子は `studio_room_id`。

### 配置プレビュー（placement preview）

選択中の定期予約について、今後 4 週間にどのレッスンが配置されるかを一覧表示する機能。代行・時間変更・プログラム変更があれば差分として示す。

### ビジネスエラー（business error）

予約 API の応答のうち、認証失効や通信障害ではなく、業務ルール上の拒否を表すもの（満席・既予約・時間外など）。

### プログラム ID（program id）

予約 API が返すプログラムの識別子。定期予約のマッチングで主キーとして使い、名前の表記揺れに振り回されないようにする。

### 冪等キー（idempotency key）

同じ定期予約の同日二重実行を防ぐための一意キー。定期予約の識別子と予定日の組で構成する。

### 月間テンプレート（monthly template）

公開月間 API から取得した、代行反映前のレッスン定義。月次と週次で自動更新する。

### 仮スケジュール（tentative schedule）

公開月間 API でも reserve API でも取得できない日（祝日特別日や翌月未配信日）について、過去 1〜3 週前に観測した実スケジュールから組み立てる暫定表示。レッスンは薄いグレーと「仮」バッジで実データと区別する。利用者は仮スケジュール上のレッスンに対しても予約予定を登録でき、開放日 09:00 に実データへ自動的に差し替わる。

### 予約 API（reservation API）

セントラルスポーツ会員向け予約サイト（reserve.central.co.jp）の内部 API。ログイン・スケジュール取得・予約操作を担う。

### 予約の由来（reservation source）

予約が単発予約と定期予約のどちらから取られたかを示す属性。単発予約のカレンダー上で、その予約が定期予約由来であることを示すのに使う。

### 予約予定（upcoming reservations）

これから定期予約によって取りにいく予定のレッスンの一覧。ダッシュボードに表示し、いつ予約を試みるかを併記する。

### 予約履歴（reservation history）

過去の自動予約の結果を時系列で並べたもの。成功・代替席での成功・失敗を含む。UI 上の呼び名は「履歴」、内部のデバッグ用ログとは区別する。

## 運用用語

### 9 時の予約実行（9-o'clock run）

予約開放の 9:00 ちょうどに、登録済み定期予約の予約 POST を自動で送る処理。設計・運用の文脈で「発火」と呼ぶこともあるが、UI 文言には使わない。

### dry-run モード（dry-run mode）

予約 POST を抑止し、内容確認だけを行う開発・検証用のフラグ。既定は OFF。

### 構造化ログ（structured log）

タイムスタンプ、リクエスト識別子、関数名、所要時間、結果などを JSON Lines 形式で出力するログ。開発者向けの内部ログで、UI 上の「予約履歴」とは別物。

### 再ログイン（re-signin）

予約 API のアクセストークンが失効した際に 1 回だけ実行するログイン処理。連続失敗時はロックアウトを避けるため停止する。

### 事前ログイン（warm-up signin）

9 時の予約実行前に、8:55 頃に事前にログインして対象レッスンを解決しておく処理。Discord に準備完了を通知する。

### 週次差分取得（weekly diff sync）

月間テンプレートの更新（代行や時間変更）を取り込むため、週に 1 回差分を再取得するジョブ。

### 手動の予約実行（manual run-now）

定期予約を選んで、利用者が任意のタイミングで予約 POST を実行する操作。検証やリカバリの用途。

### 月次全件取得（monthly full sync）

月初に公開月間 API から翌月分のスケジュールテンプレートを取得してキャッシュするジョブ。

### 直列化キュー（write queue）

スケジューラと UI からの永続ストアへの書き込み競合を避けるため、単一のキューで書き込みを順序化する仕組み。

### 当日サマリー（today's summary）

ダッシュボードの先頭に表示する横長のバー。今朝の予約結果を件数と明細でまとめる。翌朝の実行開始時にリセットされる。

### 冪等猶予時間（misfire grace time）

スケジューラが本来の発火時刻を逃した場合に、どの程度の遅延まで再発火を許容するかの設定値。

### マスキング境界（masking boundary）

ログ、通知、画面応答に値を出力する直前で、メールアドレス・パスワード・トークン・Cookie 等を `***` に置換する処理の集中実装箇所。

## 識別子・外部依存

### アクセストークン（access token）

予約 API のログイン成功時に Set-Cookie で返される認証トークン。プロセス内メモリだけで保持する。

### シークレットストア（secret store）

会員メールアドレスとパスワードを Fernet で暗号化して保持する外部ボリューム。読み取り専用でマウントする。

### 店舗識別子（studio identifier）

予約 API の `studio_id` と `studio_room_id`、公開月間 API の `club_code` と `sisetcd` の組。両者の対応はアプリ側で持つ。

### デバイス ID（device id）

予約 API が初回訪問時に Cookie として要求する 40 文字のランダム hex 値。ブラウザ互換のためクライアント側で生成する。

### hacomono

セントラルスポーツの予約サイトが採用しているフィットネス施設管理 SaaS。予約 API の挙動は hacomono の仕様に準ずる。

### リクエスト識別子（request id）

外部 API 呼び出しごとに付与する一意識別子。実行履歴で呼び出しを相関付けるために使う。
