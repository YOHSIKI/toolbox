## ドメイン用語

### 9 時発火（9-o'clock trigger）

予約ターゲットに登録したレッスンを毎朝 9:00 ちょうどに自動で予約 POST する仕組み。

### インストラクター代行（instructor substitution）

通常担当ではなく別の担当者がレッスンを受け持つこと。週によって発生しうるため、自動予約のキーには含めない。

### カレンダー UI（calendar UI）

月間テンプレートと直近 1 週間の実スケジュールを重ねて曜日×時刻のマスで表示する画面。

### 希望席（seat preference）

自動予約ターゲットが発火時に試行する座席番号の優先順位付き配列。第 1 候補から順に試行する。

### 公開月間 API（public monthly API）

店舗の月間スケジュールを JSONP 形式で返す外部サイトのエンドポイント。代行反映前のテンプレートを提供する。

### 座席番号（seat number / no）

レッスンごとに割り当てられた個別の席識別子。座席レイアウトによって範囲が異なる（35 名・20 名・25 名）。

### 座席レイアウト（seat layout / studio room space）

同じスタジオルーム内で複数定義された座席配置（定員違い）。レッスンごとにどのレイアウトを使うかが切り替わる。

### 実スケジュール（actual schedule）

予約 API から取得した直近 1 週間分のレッスン一覧。代行やその他の変更が反映された状態。

### 自動予約ターゲット（reservation target）

曜日・時刻・プログラム名パターン・希望席の配列などをキーとした自動予約の定義。

### 準備ジョブ（warm-up job）

9 時発火前の 8:55 頃に実行する事前処理。ログインと対象レッスンの解決を済ませ、Discord に準備完了を通知する。

### 席変更（seat move）

既存予約の座席番号を別の空き番号に変更する操作。

### スケジュールキャッシュ（schedule cache）

公開月間 API と予約 API から取得したスケジュールを永続ストアに一時保存したデータ。TTL で再取得する。

### 月間テンプレート（monthly template）

公開月間 API から取得した代行反映前のレッスン定義。月次と週次で自動更新する。

### 店舗（studio）

セントラルスポーツのクラブを表す概念。識別子は `studio_id`。

### 店舗ルーム（studio room）

店舗内のスタジオ・プール・ジム等を区別する単位。識別子は `studio_room_id`。

### ビジネスエラー（business error）

予約 API の応答のうち、認証失効や通信障害ではなく、業務ルール上の拒否を表すもの（満席・既予約・時間外など）。

### プログラム名パターン（program name pattern）

自動予約ターゲットがマッチングに使う、プログラム名の部分一致または正規表現。同一プログラムが時刻違いで複数並ぶ場合は時刻と組み合わせる。

### 冪等キー（idempotency key）

同一ターゲットの同日二重発火を排他するための一意キー。ターゲット識別子と予定日の組で構成する。

### 予約 API（reservation API）

セントラルスポーツ会員向け予約サイト（reserve.central.co.jp）の内部 API。ログイン・スケジュール・予約操作を担う。

## 運用用語

### dry-run モード（dry-run mode）

予約 POST を抑止し、内容確認のみ行う開発・検証用のフラグ。デフォルトは OFF。

### 構造化ログ（structured log）

タイムスタンプ・リクエスト識別子・関数名・所要時間・結果などを JSON Lines 形式で出力するログ。

### マスキング境界（masking boundary）

ログ・通知・UI 応答に値を出力する直前で、メール・パスワード・トークン・Cookie 等を `***` に置換する処理の集中実装箇所。

### 冪等猶予時間（misfire grace time）

スケジューラが本来の発火時刻を逃した場合に、どの程度の遅延まで再発火を許容するかの設定値。

### 再ログイン（re-signin）

予約 API のアクセストークンが失効した際に 1 回だけ再実行するログイン処理。連続失敗時はロックアウト回避のため停止する。

### 週次差分取得（weekly diff sync）

月間テンプレートの更新（代行や枠変更）を取り込むため、週 1 回差分を再取得するジョブ。

### 直列化キュー（write queue）

スケジューラと UI からの永続ストアへの書き込み競合を避けるため、単一のキューで書き込みを順序化する仕組み。

### 手動即時発火（manual run-now）

自動予約ターゲットを選んで、利用者が任意のタイミングで予約 POST を実行する操作。検証・リカバリ用途。

### 月次全件取得（monthly full sync）

月初に公開月間 API から翌月分のスケジュールテンプレートを取得してキャッシュするジョブ。

## 識別子・外部依存

### アクセストークン（access token）

予約 API のログイン成功時に Set-Cookie で返される認証トークン。プロセス内メモリでのみ保持する。

### デバイス ID（device id）

予約 API が初回訪問時に Cookie として要求する 40 文字のランダム hex 値。ブラウザ互換のためクライアント側で生成する。

### hacomono

セントラルスポーツの予約サイトが採用しているフィットネス施設管理 SaaS。内部 API の挙動は hacomono の仕様に準ずる。

### シークレットストア（secret store）

会員メールアドレスとパスワードを Fernet で暗号化して保持する外部ボリューム。読み取り専用でマウントする。

### 店舗識別子（studio identifier）

予約 API の `studio_id`・`studio_room_id` と、公開月間 API の `club_code`・`sisetcd` の組。両者の対応はアプリ側で持つ。

### リクエスト識別子（request id）

外部 API 呼び出しごとに付与する一意識別子。実行ログで呼び出しを相関付けるために使う。
